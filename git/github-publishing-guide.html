1

Key Concepts — What Are We Actually Doing?

Before touching anything, let's build a mental model so nothing feels mysterious.

You're about to do something that professional software developers do every day. The good news is the core ideas are simple, and you only need to understand three of them.

Git
A program that tracks changes to files over time. Think of it like "Track Changes" in Word, but for any file on your computer. It keeps a full history of every version of your work.
GitHub
A website that stores your Git-tracked files online — like a cloud backup. It also has a free feature that turns your files into a live website anyone can visit.
Repository (repo)
A project folder that Git is keeping track of. Your folder on your computer is the "local repo." The copy on GitHub is the "remote repo."
Commit
A saved snapshot of your files at a specific moment. Like saving a document, but permanent. You can always go back to any previous commit.
Push
Sending your commits from your computer up to GitHub. This is how the online copy stays up to date.
GitHub Pages
A free GitHub feature that reads the HTML files in your repo and serves them as a real website — at a public address anyone can open in a browser.
💡
The flow in plain English You create files on your computer → Git tracks them → you "commit" a snapshot → you "push" to GitHub → GitHub Pages shows them as a website.
2

Creating a GitHub Account

Free — no credit card required.

✅
If you already have a GitHub account, skip to Section 3.
Go to GitHub.com Open your web browser and go to https://github.com. Click the big Sign up button.
Enter your email and create a password Use an email address you check regularly — you'll need to confirm it.
Choose a username This will be part of your website's address, so choose something you like. For example, if you choose jsmith, your site will be at jsmith.github.io.
Verify your email GitHub will send a confirmation email. Click the link in it to activate your account.
💡
Tip on usernames Keep it professional — your GitHub username is often visible to employers or collaborators. Avoid spaces (use hyphens instead) and numbers at the start.
3

Choosing and Installing Your Tools

You have two options — a visual point-and-click app, or typed commands. Both work great.

Git was invented by programmers for programmers, so it was originally used entirely by typing commands. But today there are excellent visual apps that let you do everything with mouse clicks instead. Either path gets you to the same destination.

🖥️ GitHub Desktop (recommended for beginners)
  • No typing commands
  • Shows you exactly what changed
  • Includes Git automatically
  • Free — made by GitHub
  • Windows & Mac
⌨️ Command Line (Git Bash / Terminal)
  • Type commands to do everything
  • Works on any computer
  • Used by professionals worldwide
  • More control and flexibility
  • All platforms

The rest of this guide will show both methods side by side — choose the one that feels comfortable to you. You can always switch later.

Installing GitHub Desktop (GUI option)

Download GitHub Desktop Go to https://desktop.github.com and click the Download button. It will detect your operating system automatically.
Run the installer On Windows, run the downloaded .exe file. On Mac, open the .dmg and drag the app to your Applications folder.
Sign in to your GitHub account When GitHub Desktop opens, click Sign in to GitHub.com and enter the credentials you just created.
Configure your identity GitHub Desktop will ask for your name and email. These are attached to your commits so you know who made each change. Use your real name.

Installing Git for the Command Line (CLI option)

Windows: Download Git for Windows Go to https://git-scm.com/download/win. The download will start automatically. Run the installer and accept all defaults — this gives you Git Bash, a command line app that understands Git.
Mac: Git may already be installed Open Terminal (press Cmd+Space and type "Terminal"). Type git --version and press Enter. If you see a version number, you're done. If not, Mac will prompt you to install it automatically.
Tell Git who you are Open Git Bash (Windows) or Terminal (Mac) and type these two commands, pressing Enter after each. Replace the quoted text with your own name and email:
Git Bash / Terminal
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
You only need to do this once. Git will use this identity for all future commits.
ℹ️
What is Git Bash? Git Bash is a special command-line window that understands both Git commands and standard file commands. It's what you'll open when this guide says "open a terminal."
4

Setting Up Your Project Folder

Creating the folder that will hold your website files.

Your website will live in a regular folder on your computer. The only rule is that its name should have no spaces (use hyphens instead). For this guide, we'll call the folder my-website and place it in Documents.

In GitHub Desktop, you don't need to create the folder manually — you can do it all from within the app in Section 5. Just decide where you want it (we'll use Documents) and move on.

💡
If you want to create the folder first, open File Explorer (Windows) or Finder (Mac), navigate to Documents, right-click in an empty area, and select New Folder. Name it my-website.

Open Git Bash (Windows) or Terminal (Mac) and type these commands:

Git Bash / Terminal
cd ~/Documents
mkdir my-website
cd my-website

cd means "change directory" (navigate to a folder). mkdir means "make directory" (create a folder). After these commands, you are inside the new folder.

5

Making It a Git Repository

"Initializing" a repo means telling Git to start tracking this folder.

Right now, my-website is just a regular folder. We need to tell Git: "watch this folder." That one-time setup is called initializing a repository.

Open GitHub Desktop and click "Create a New Repository on your hard drive" You'll see this option on the welcome screen, or under File → New Repository.
Fill in the form
  • Name: my-website
  • Local path: Click "Choose..." and navigate to your Documents folder
  • Initialize this repository with a README: check this box
Click "Create Repository" GitHub Desktop creates the folder and sets up Git inside it. You'll see the repository appear in the left panel.

Make sure you are inside the my-website folder (you should be if you followed Section 4), then run:

Git Bash / Terminal
git init
git branch -M main

git init creates a hidden .git folder inside your directory. That folder is where Git stores all its history — don't touch or delete it. git branch -M main sets the primary branch name to "main" (the modern standard).

⚠️
Hidden files Git creates a hidden folder called .git inside your project. On Windows you may not see it in File Explorer by default — that's fine. Never delete it, or you'll lose Git's tracking data.
6

Adding Your Web Page

Creating the HTML file that will become your website's home page.

GitHub Pages looks for a file named exactly index.html in your repository. That file becomes your home page. Let's create a simple one now.

ℹ️
What text editor should I use? Notepad (Windows) or TextEdit (Mac) work fine for simple HTML. For a better experience, download Visual Studio Code (free, at https://code.visualstudio.com) — it colors your code and helps catch mistakes. Do NOT use Word or Google Docs — they add hidden formatting that breaks HTML.

Creating index.html

Open your text editor, create a new file, and paste the following. Then save it as index.html inside your my-website folder.

<!-- This is your home page -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>
</head>
<body>
  <h1>Welcome to My Website!</h1>
  <p>This page is hosted on GitHub Pages.</p>
</body>
</html>
💡
Saving tip (Notepad on Windows) When saving in Notepad, set Save as type to "All Files" and manually type index.html as the filename. If you leave it as "Text Document," Windows will save it as index.html.txt, which will not work as a web page.

Committing your file (saving it to Git history)

Just saving the file on your computer isn't enough — you also need to tell Git to take a snapshot of it. This is called making a commit.

Return to GitHub Desktop You'll see your new index.html listed in the left panel under "Changes." A green + icon shows it's a new file.
Write a commit message At the bottom left, there's a text box that says "Summary (required)." Type a short description like: Add home page
Click "Commit to main" The file is now saved in Git's history. The Changes panel will empty — that means everything is committed.
Git Bash / Terminal
git add index.html
git commit -m "Add home page"

git add "stages" the file — tells Git you want to include it in the next snapshot. git commit -m takes that snapshot with a message describing it. The message is for your own reference — write something that makes sense to you.

7

Publishing to GitHub

Sending your files from your computer up to GitHub's servers.

Your commits are saved locally on your computer. Now we need to create a repository on GitHub.com and "push" your files up to it.

Click "Publish repository" in the top bar GitHub Desktop shows this button when your repo hasn't been pushed to GitHub yet.
Review the settings
  • Name: Keep it as my-website
  • Keep this code private: Uncheck this — it must be public for GitHub Pages to work
Click "Publish Repository" GitHub Desktop creates the repo on GitHub and uploads your files. When it finishes, you can click View on GitHub to see it online.

First, go to https://github.com/new and create a new repository named my-website. Set it to Public. Do not check "Initialize with README." Then copy the HTTPS URL GitHub shows you.

Back in your terminal:

Git Bash / Terminal
# Replace YOUR_USERNAME with your actual GitHub username
git remote add origin https://github.com/YOUR_USERNAME/my-website.git
git push -u origin main

remote add origin tells Git where GitHub is ("origin" is the standard nickname for your GitHub remote). push -u origin main sends your commits up. The -u flag remembers the destination so future pushes only need git push.

⚠️
Authentication prompt Git may open a browser window asking you to sign in to GitHub. Complete the sign-in there, and the push will continue automatically. This is a one-time setup.
✅
Verify it worked Open a browser and go to https://github.com/YOUR_USERNAME/my-website. You should see your index.html file listed there. Your files are now safely backed up online!
8

Turning It Into a Live Website

Enabling GitHub Pages — the free hosting that makes your files into a real website.

Right now your files are on GitHub, but they're stored as code. The final step is to flip a switch that tells GitHub to serve those files as a website.

Open your repository on GitHub.com Go to https://github.com/YOUR_USERNAME/my-website.
Click "Settings" (the gear icon tab near the top right) This opens the settings for your repository — not your account settings.
Scroll down the left sidebar and click "Pages" This is under the "Code and automation" section.
Under "Branch," click the dropdown that says "None" and select "main" Leave the folder as / (root). Then click Save.
Wait about 1–2 minutes, then refresh the page A blue banner will appear with your site's address. It will look like:
https://YOUR_USERNAME.github.io/my-website/
Click the link and see your live website! Share that address with anyone — they can open it in any browser, anywhere in the world.
💡
Deployment takes a moment After clicking Save, GitHub has to "build" your site. It usually takes 30–90 seconds. If the link shows a 404 error at first, just wait a minute and refresh.
9

Making Updates Later

The everyday workflow once everything is set up — much simpler than the first time.

From now on, whenever you want to update your website, the process is: edit → commit → push. That's it. GitHub Pages will automatically update your live site within a minute or two.

Edit your files Open and edit any file in your my-website folder using your text editor. Save the file normally.
Switch to GitHub Desktop It will automatically detect the changes and show them in the Changes panel.
Write a commit message and click "Commit to main" Describe what you changed, e.g., "Update home page heading"
Click "Push origin" (top bar) This sends your commit to GitHub. Your live site will update automatically in about a minute.

Navigate to your project folder and run:

Git Bash / Terminal
cd ~/Documents/my-website
git add -A
git commit -m "Describe what you changed"
git push

git add -A stages all changed and new files at once (quicker than naming them individually). After git push, GitHub receives your changes and Pages rebuilds the site automatically.

Adding more pages

To add a second page (say, an "About" page), just create a file called about.html in the same folder and link to it from your index.html with:

<a href="about.html">About Me</a>

Then commit and push as usual. It will be live at https://YOUR_USERNAME.github.io/my-website/about.html.

Adding images

Copy your image file (e.g. photo.jpg) into your my-website folder. Reference it in your HTML like this:

<img src="photo.jpg" alt="A description of the photo">

Include the image in your next commit and push, and it will appear on your live site.

  • Created the project folder in Documents
  • Initialized a Git repository in the folder
  • Created index.html and saved it in the folder
  • Committed the file with a descriptive message
  • Created a public repository on GitHub.com
  • Pushed the local repo to GitHub
