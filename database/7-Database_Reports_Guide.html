Database Reports & SQL Server Reporting Services

📚 Module Overview

Database reporting is essential for transforming raw data into actionable insights. This module covers the fundamentals of database reporting, explores various reporting tools, and provides hands-on guidance for creating professional reports using Microsoft SQL Server Reporting Services (SSRS) Report Builder. You'll learn to create tabular reports, matrix reports, charts, parameterized reports, and more.

What is Database Reporting?

📖 Definition

Database reporting is the process of extracting, transforming, and presenting data from databases in a structured, readable format. Reports turn raw data into meaningful information that supports decision-making.

Types of Reports

1. Operational Reports

Purpose: Day-to-day business operations

  • Examples: Daily sales report, inventory status, customer orders
  • Frequency: Daily, hourly, or real-time
  • Audience: Operational staff, managers
  • Characteristics: Detail-oriented, current data, specific metrics

2. Analytical Reports

Purpose: Analyze trends and patterns

  • Examples: Sales trends by region, customer segmentation, year-over-year comparison
  • Frequency: Weekly, monthly, quarterly
  • Audience: Managers, analysts, executives
  • Characteristics: Historical data, aggregations, visualizations

3. Tactical Reports

Purpose: Support specific decisions or actions

  • Examples: Product performance analysis, marketing campaign ROI
  • Frequency: As needed
  • Audience: Department managers, project teams
  • Characteristics: Focused scope, actionable insights

4. Strategic Reports

Purpose: High-level business intelligence

  • Examples: Executive dashboard, KPI scorecards, market analysis
  • Frequency: Monthly, quarterly, annually
  • Audience: Executives, board members
  • Characteristics: Summary-level, trends, forecasts

Key Components of Effective Reports

📋 Clear Structure

Logical organization with headers, sections, and groupings

📊 Appropriate Visualization

Charts, tables, or matrices that match the data type

🎯 Relevant Filters

Parameters that let users focus on specific data

📈 Key Metrics

Important calculations, totals, and KPIs

🎨 Professional Design

Consistent formatting, readable fonts, appropriate colors

⏱️ Good Performance

Efficient queries that run quickly

Popular Reporting Tools

Microsoft SQL Server Reporting Services (SSRS)

Type: Enterprise reporting platform (focus of this course)

✅ Strengths:

  • Integrated with SQL Server
  • Free with SQL Server license
  • Powerful Report Builder tool
  • Paginated reports for printing
  • Subscription and scheduling capabilities
  • Multiple export formats (PDF, Excel, Word)
  • Parameterized and drill-through reports

⚠️ Considerations:

  • Windows-based (though Linux support added)
  • Steeper learning curve than some alternatives
  • Requires SQL Server infrastructure

Best For: Organizations using SQL Server, need for complex paginated reports, enterprise reporting requirements

Microsoft Power BI

Type: Modern business intelligence and visualization platform

✅ Strengths:

  • User-friendly drag-and-drop interface
  • Beautiful, interactive visualizations
  • Cloud-based with collaboration features
  • Connects to many data sources
  • Real-time dashboards
  • Natural language queries (Q&A feature)
  • Mobile apps

Best For: Interactive dashboards, self-service BI, exploratory analysis, real-time monitoring

Crystal Reports

Type: Traditional reporting tool (SAP)

✅ Strengths:

  • Long-established (since 1991)
  • Excellent for complex layouts
  • Strong formatting capabilities
  • Multiple data source support

Best For: Legacy systems, organizations already using SAP products

Tableau

Type: Data visualization and BI platform

✅ Strengths:

  • Exceptional visualization capabilities
  • Intuitive interface
  • Strong data exploration features
  • Large community and resources

Best For: Data exploration, interactive dashboards, visual analytics

Jaspersoft (JasperReports)

Type: Open-source reporting (Java-based)

✅ Strengths:

  • Open-source option available
  • Cross-platform (Java)
  • Good for embedded reporting
  • Active community

Best For: Java environments, cost-conscious projects, embedded reporting needs

What is Report Builder?

Report Builder is a free report authoring tool for creating paginated reports that can be published to SQL Server Reporting Services. It provides a user-friendly interface for designing professional reports without extensive programming knowledge.

Key Concepts

Data Sources

Connection information to your database

  • Connection String: Server and database location
  • Credentials: How to authenticate
  • Types: SQL Server, Oracle, OLE DB, ODBC, etc.

Datasets

Queries that retrieve data from data sources

  • Query: SQL statement or stored procedure
  • Fields: Columns returned by the query
  • Parameters: Variables passed to the query

Report Items

Visual elements added to the report

  • Tablix: Table or matrix for displaying data
  • Chart: Bar, line, pie, etc.
  • Text Box: Static text or expressions
  • Image: Logos or embedded images
  • Rectangle: Container for other items

Tutorial: Simple Sales Report

🎯 Goal

Create a basic tabular report showing product sales data with totals and formatting.

1 Create New Report
  1. Launch Report Builder
  2. Click Blank Report (or File → New)
  3. You'll see an empty design surface
2 Create Data Source
  1. In the Report Data pane, right-click Data Sources
  2. Click Add Data Source
  3. Enter a name: SalesDB
  4. Select connection type: Microsoft SQL Server
  5. Click Build to configure connection:
    • Server name: localhost or your SQL Server instance
    • Authentication: Windows or SQL Server
    • Database: Select your database
  6. Click OK to test and save
💡 Tip: Use Test Connection to verify before saving.
3 Create Dataset
  1. Right-click Datasets in Report Data pane
  2. Click Add Dataset
  3. Enter name: SalesData
  4. Select data source: SalesDB
  5. Query type: Text
  6. Enter SQL query:
SELECT p.ProductName, c.CategoryName, SUM(od.Quantity) AS TotalQuantity, SUM(od.Quantity * od.UnitPrice) AS TotalSales FROM OrderDetails od INNER JOIN Products p ON od.ProductID = p.ProductID INNER JOIN Categories c ON p.CategoryID = c.CategoryID GROUP BY p.ProductName, c.CategoryName ORDER BY c.CategoryName, TotalSales DESC;
  1. Click Refresh Fields to load field list
  2. Click OK
✅ Result: You should now see the dataset with fields in the Report Data pane.
4 Insert Table
  1. On the Insert tab, click Table → Insert Table
  2. Click on the design surface to place table
  3. A 3-column table appears with header and data rows
5 Add Fields to Table
  1. From Report Data pane, drag CategoryName to first column
  2. Drag ProductName to second column
  3. Drag TotalQuantity to third column
  4. Right-click on table, select Insert Column → Right
  5. Drag TotalSales to the new fourth column

The table automatically creates expressions like =Fields!CategoryName.Value

6 Format the Report

Format Headers:

  1. Click the header row
  2. In Properties: Set BackgroundColor to DarkBlue
  3. Set Color to White
  4. Set FontWeight to Bold

Format Currency:

  1. Click the TotalSales data cell (not header)
  2. In Properties: Set Format to C2 (currency with 2 decimals)

Format Numbers:

  1. Click TotalQuantity data cell
  2. Set Format to N0 (number with no decimals)

Adjust Column Widths:

  1. Click and drag column borders to adjust width
  2. Make ProductName column wider
  3. Make numeric columns narrower
7 Add Totals
  1. Right-click on TotalQuantity data cell
  2. Select Add Total
  3. A footer row is added with =Sum(Fields!TotalQuantity.Value)
  4. Repeat for TotalSales column
  5. In the footer, first column, type "Total:"
8 Add Report Title
  1. On Insert tab, click Text Box
  2. Click above the table to place it
  3. Type: "Product Sales Report"
  4. Select the text, then in Home tab:
    • Set font size to 18pt
    • Make it bold
    • Center align
9 Preview the Report
  1. Click the Run button on the ribbon (or F5)
  2. The report executes and displays data
  3. Verify that:
    • Data appears correctly
    • Formatting is applied
    • Totals are calculated
  4. Click Design button to return to design view
10 Save the Report
  1. Click File → Save (or Ctrl+S)
  2. Choose location (local file or report server)
  3. Enter filename: ProductSalesReport.rdl
  4. Click Save

🎉 Congratulations!

You've created your first SSRS report! The .rdl file can be opened in Report Builder or deployed to a report server.

1. Tabular Reports

Basic Table Report

Use Case: Display data in rows and columns with grouping

Features:

  • Row grouping by category
  • Subtotals for each group
  • Grand totals at the bottom
  • Alternating row colors for readability

Example: Employee Roster by Department

SELECT d.DepartmentName, e.LastName + ', ' + e.FirstName AS EmployeeName, e.JobTitle, e.HireDate, e.Salary FROM Employees e INNER JOIN Departments d ON e.DepartmentID = d.DepartmentID ORDER BY d.DepartmentName, e.LastName;

Implementation Tips:

  • Add row group on DepartmentName
  • Insert group header with department name
  • Add group footer with department subtotals
  • Use expression for alternating colors: =IIf(RowNumber(Nothing) Mod 2 = 0, "White", "LightGray")

2. Matrix Reports (Crosstab)

Matrix/Pivot Table Report

Use Case: Show data in rows and columns with aggregations

Features:

  • Dynamic columns based on data
  • Cross-tabulated values
  • Row and column subtotals

Example: Sales by Region and Quarter

SELECT r.RegionName, DATEPART(QUARTER, o.OrderDate) AS Quarter, YEAR(o.OrderDate) AS Year, SUM(od.Quantity * od.UnitPrice) AS Sales FROM Orders o INNER JOIN OrderDetails od ON o.OrderID = od.OrderID INNER JOIN Customers c ON o.CustomerID = c.CustomerID INNER JOIN Regions r ON c.RegionID = r.RegionID WHERE YEAR(o.OrderDate) = YEAR(GETDATE()) GROUP BY r.RegionName, DATEPART(QUARTER, o.OrderDate), YEAR(o.OrderDate);

Creating a Matrix:

  1. Insert → Matrix
  2. Drag RegionName to Row Groups
  3. Drag Quarter to Column Groups
  4. Drag Sales to Data (aggregated by Sum)
  5. Right-click row/column groups to add totals

3. Chart Reports

Visualization with Charts

Chart Types Available:

  • Column/Bar: Compare values across categories
  • Line: Show trends over time
  • Pie/Doughnut: Show proportions
  • Area: Show cumulative totals
  • Scatter: Show correlation between values

Example: Monthly Sales Trend (Line Chart)

SELECT DATEPART(YEAR, OrderDate) AS Year, DATEPART(MONTH, OrderDate) AS Month, DATENAME(MONTH, OrderDate) AS MonthName, SUM(od.Quantity * od.UnitPrice) AS MonthlySales FROM Orders o INNER JOIN OrderDetails od ON o.OrderID = od.OrderID WHERE OrderDate >= DATEADD(MONTH, -12, GETDATE()) GROUP BY DATEPART(YEAR, OrderDate), DATEPART(MONTH, OrderDate), DATENAME(MONTH, OrderDate) ORDER BY Year, Month;

Creating a Chart:

  1. Insert → Chart → Choose Chart Type (Line)
  2. Click on chart to open Chart Data pane
  3. Drag MonthName to Category Groups
  4. Drag MonthlySales to Values
  5. Format chart:
    • Add chart title
    • Format axis labels
    • Add data labels if desired
    • Adjust colors and styles

4. Parameterized Reports

Reports with User Input

Use Case: Allow users to filter or customize reports

Example: Sales Report with Date Range

Step 1: Create Parameters

  1. In Report Data pane, right-click Parameters
  2. Add Parameter: StartDate
    • Data type: Date/Time
    • Default value: =DateAdd("d", -30, Today())
  3. Add Parameter: EndDate
    • Data type: Date/Time
    • Default value: =Today()

Step 2: Use Parameters in Query

SELECT o.OrderDate, c.CustomerName, SUM(od.Quantity * od.UnitPrice) AS OrderTotal FROM Orders o INNER JOIN Customers c ON o.CustomerID = c.CustomerID INNER JOIN OrderDetails od ON o.OrderID = od.OrderID WHERE o.OrderDate >= @StartDate AND o.OrderDate <= @EndDate GROUP BY o.OrderDate, c.CustomerName ORDER BY o.OrderDate;

Advanced: Cascading Parameters

Create dependent parameters (e.g., Category → Product):

  1. Create Category parameter with dataset listing categories
  2. Create Product parameter with dataset filtered by selected category
  3. Dataset query uses WHERE CategoryID = @CategoryID

5. Drill-Through Reports

Linked Detail Reports

Use Case: Summary report that links to detailed report

Implementation:

  1. Create main summary report (e.g., Sales by Category)
  2. Create detail report (e.g., Products in Category)
  3. In summary report:
    • Right-click the field to make clickable
    • Properties → Action
    • Select "Go to report"
    • Choose detail report
    • Map parameters (e.g., CategoryID)
  4. User clicks value in summary → detail report opens

6. Subreports

Embedded Reports

Use Case: Include one report inside another

Example: Customer Report with Order History

  1. Create main report showing customer details
  2. Create subreport showing orders
  3. In main report:
    • Insert → Subreport
    • Properties → Select subreport
    • Set parameters (e.g., CustomerID)
⚠️ Performance Note: Subreports execute separately for each row, which can be slow. Consider using tables with grouping instead.

Expressions and Calculated Fields

📐 Report Builder Expressions

Expressions use Visual Basic syntax and start with =

Common Expression Examples

1. Concatenating Text:

=Fields!FirstName.Value & " " & Fields!LastName.Value

2. Conditional Formatting:

=IIf(Fields!Sales.Value > 10000, "Green", "Red")

3. Date Formatting:

=Format(Fields!OrderDate.Value, "MM/dd/yyyy")

4. Calculations:

=Fields!Quantity.Value * Fields!UnitPrice.Value

5. Percentage Calculation:

=(Fields!Sales.Value / Sum(Fields!Sales.Value, "DataSet1")) * 100

6. Running Total:

=RunningValue(Fields!Sales.Value, Sum, Nothing)

7. Row Number:

=RowNumber(Nothing)

8. Alternating Row Colors:

=IIf(RowNumber(Nothing) Mod 2 = 0, "White", "AliceBlue")

Grouping and Sorting

Adding Row Groups

  1. Right-click a row in your table
  2. Insert Group → Parent Group
  3. Select grouping field (e.g., CategoryName)
  4. Choose to add group header/footer
  5. Add aggregations in group footer:
    • =Sum(Fields!Sales.Value)
    • =Count(Fields!ProductID.Value)
    • =Avg(Fields!Price.Value)

Interactive Sorting

Let users click column headers to sort:

  1. Select the text box in the column header
  2. Properties → Interactive Sorting
  3. Enable interactive sorting
  4. Select field to sort by

Visibility and Toggle Items

Drill-Down (Expand/Collapse)

Hide detail rows initially, expand on click:

  1. Select the row to hide (detail row)
  2. Properties → Visibility
  3. Set to "Hide"
  4. Check "Display can be toggled by this report item"
  5. Select the text box that will toggle (group header)

Result: A +/- icon appears, clicking expands/collapses details

Page Layout and Breaks

Controlling Pagination

1. Add Page Break After Group:

  1. Right-click on row group
  2. Group Properties
  3. Page Breaks tab
  4. Check "Between each instance of a group"

2. Keep Together:

  1. Select table or group
  2. Properties → KeepTogether = True

3. Repeat Headers on Each Page:

  1. Click table row/column group area
  2. Advanced Mode (arrow in grouping pane)
  3. Select static row (header)
  4. Set KeepWithGroup = After
  5. Set RepeatOnNewPage = True

Headers and Footers

Adding Page Headers/Footers

Enable Header/Footer:

  1. Insert tab → Header or Footer
  2. The header/footer section appears

Common Elements:

  • Page Numbers: =Globals!PageNumber & " of " & Globals!TotalPages
  • Report Name: =Globals!ReportName
  • Execution Time: =Globals!ExecutionTime
  • User Name: =User!UserID

Export Formats

📤 Available Export Formats:

  • PDF: Best for printing, maintains exact layout
  • Excel: Editable data, good for further analysis
  • Word: Editable document format
  • CSV: Raw data, no formatting
  • XML: Structured data format
  • MHTML: Web archive (single file)

💡 Export Optimization Tips:

  • Excel: Avoid merged cells, keep layout simple
  • PDF: Set page size appropriately, use page breaks
  • CSV: Only exports data, no formatting or charts

Performance Optimization

✅ DO: Optimize Your Queries

  • Select only needed columns, not SELECT *
  • Filter data in the query (WHERE clause), not in report
  • Use indexes on filter and join columns
  • Use stored procedures for complex logic
  • Consider creating a view for report data
-- Good: Selective query SELECT ProductName, Category, Price, Stock FROM Products WHERE IsActive = 1 AND Price > 0; -- Bad: Gets everything SELECT * FROM Products;

❌ AVOID: Subreports in Tablixes

Subreports execute for each row = very slow!

Better: Use grouping or combine queries with JOINs

✅ DO: Use Report Caching

  • Cache reports that run frequently
  • Set appropriate cache expiration
  • Use snapshots for historical reports

✅ DO: Limit Dataset Size

  • Use TOP or pagination for large datasets
  • Provide date range parameters
  • Consider summary vs detail reports
-- Limit results with TOP SELECT TOP 1000 OrderDate, CustomerName, OrderTotal FROM Orders ORDER BY OrderDate DESC;

Design Principles

✅ DO: Keep It Simple

  • Focus on key information
  • Avoid cluttered layouts
  • Use white space effectively
  • Limit to 5-7 colors

✅ DO: Choose Appropriate Visualizations

  • Tables: Precise values, detailed data
  • Bar Charts: Comparing categories
  • Line Charts: Trends over time
  • Pie Charts: Proportions (use sparingly, max 5-7 slices)

❌ AVOID: Misleading Visualizations

  • Truncated Y-axis that exaggerates differences
  • 3D charts that distort perception
  • Too many pie slices (use bar chart instead)

✅ DO: Format Numbers Consistently

  • Use currency symbols: C2
  • Use thousands separators: N0
  • Consistent decimal places
  • Align numbers right

Usability

✅ DO: Provide Clear Context

  • Descriptive report title
  • Include run date/time
  • Show parameter values used
  • Add legends for charts
  • Include units of measure

✅ DO: Make Reports Interactive

  • Add drill-down for details
  • Enable sorting by clicking headers
  • Provide meaningful parameters
  • Use drill-through to related reports

✅ DO: Test Thoroughly

  • Test with various parameter combinations
  • Test with empty result sets
  • Test with maximum expected data volume
  • Verify calculations manually
  • Test all export formats
  • Get user feedback

Maintenance

✅ DO: Document Your Reports

  • Add description in report properties
  • Comment complex expressions
  • Document data sources and refresh schedules
  • Maintain change log

✅ DO: Use Consistent Naming

  • Clear dataset names: SalesData, EmployeeList
  • Descriptive parameter names: StartDate, not Param1
  • Standard prefixes for groups, totals

Security

✅ DO: Implement Proper Security

  • Use Windows Authentication when possible
  • Don't hardcode credentials
  • Implement row-level security for sensitive data
  • Use parameters to filter by user context
  • Set appropriate folder permissions on report server

Troubleshooting Common Issues

Issue: Report is Slow

Solutions:

  • Optimize SQL query (check execution plan)
  • Add indexes to database tables
  • Reduce dataset size with filters
  • Remove subreports if possible
  • Enable report caching

Issue: #Error in Expression

Solutions:

  • Check for null values: Use IsNothing()
  • Verify field names match dataset
  • Check data types (can't divide strings)
  • Use IIf to handle nulls: =IIf(IsNothing(Fields!Value), 0, Fields!Value)

Issue: Report Won't Deploy

Solutions:

  • Verify Report Server URL is correct
  • Check permissions on report server
  • Ensure report server is running
  • Test connection to report server

Issue: Parameters Not Working

Solutions:

  • Verify parameter names match in query
  • Check parameter data types
  • Ensure parameters are passed to subreports
  • Test dataset query separately
