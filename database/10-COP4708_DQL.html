DQL Overview — The Language of Questions

Foundation
🔍

Foundation · What is DQLWhat SELECT Does and How Queries Are Processed

Definition — Data Query Language (DQL)

DQL is the subset of SQL used to retrieve data from a database. It consists of a single command — SELECT — which can be combined with clauses like FROM, WHERE, ORDER BY, GROUP BY, HAVING, and JOIN to answer virtually any question about the data. DQL never modifies data; it only reads it.

The SELECT Statement — Clause Anatomy

T-SQL · Full SELECT Statement — All Clauses in Execution Order
SELECT   column_list          -- 5. Which columns (or expressions) to return
FROM     table_name           -- 1. Which table(s) to read from
[JOIN    other_table ON condition]  -- 2. Combine rows from multiple tables
[WHERE   condition]           -- 3. Filter individual rows
[GROUP BY column]            -- 4. Collapse rows into groups
[HAVING  group_condition]    -- 5. Filter groups (like WHERE for groups)
[ORDER BY column ASC|DESC]  -- 6. Sort the final results
[OFFSET  n ROWS FETCH NEXT m ROWS ONLY]  -- 7. Pagination
⚙️
Write Order vs. Execution Order SQL clauses are written in a specific order (SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY), but the database engine executes them in a different order: FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY. This explains why you can't reference a SELECT alias in a WHERE clause — WHERE runs before SELECT assigns the alias.

ORDER BY and DISTINCT

Outcomes 3 & 4
🔀

Outcomes 3 & 4 · Sorting & DeduplicationORDER BY — Sorting Results; DISTINCT — Eliminating Duplicates

ORDER BY — Sorting Query Results

ORDER BY is always the last clause in a SELECT statement. Without it, SQL Server makes no guarantee about row order — results may appear in any sequence.

T-SQL · ORDER BY — All Variations
-- Sort ascending (default — A→Z, lowest→highest)
SELECT LastName, FirstName, GPA
FROM   Student
ORDER BY LastName ASC;   -- ASC is the default; can be omitted

-- Sort descending (Z→A, highest→lowest)
SELECT LastName, GPA
FROM   Student
ORDER BY GPA DESC;

-- Multi-column sort — primary sort first, then tiebreaker
SELECT DeclaredMajor, LastName, GPA
FROM   Student
ORDER BY DeclaredMajor ASC, GPA DESC;
-- Groups by major alphabetically; within each major, highest GPA first

-- Sort by column position number (column 3 = GPA)
SELECT LastName, FirstName, GPA
FROM   Student
ORDER BY 3 DESC;   -- works but avoid — fragile if columns reordered

-- Sort using a SELECT alias
SELECT LastName, CONCAT(FirstName, ' ', LastName) AS FullName, GPA
FROM   Student
ORDER BY FullName ASC;   -- ORDER BY CAN use aliases (unlike WHERE)

DISTINCT — Removing Duplicate Rows

DISTINCT is placed immediately after SELECT and removes duplicate rows from the result. It applies to the entire row returned, not just one column.

T-SQL · DISTINCT — Remove Duplicate Values
-- Without DISTINCT — returns a DeclaredMajor for every student row (with repeats)
SELECT DeclaredMajor
FROM   Student;
-- Returns: CIS, MIS, CIS, CS, MIS, CIS, CS, MIS  (8 rows)

-- With DISTINCT — returns each unique value only once
SELECT DISTINCT DeclaredMajor
FROM   Student;
-- Returns: CIS, MIS, CS  (3 rows)

-- DISTINCT on multiple columns — combination must be unique
SELECT DISTINCT DeclaredMajor, IsEnrolled
FROM   Student;
-- Returns distinct (Major, IsEnrolled) pairs

-- Practical use: find all departments that have at least one instructor
SELECT DISTINCT DeptID
FROM   Instructor
ORDER BY DeptID;
💡
DISTINCT vs. GROUP BY DISTINCT and GROUP BY with no aggregation produce the same result, but GROUP BY is more powerful because you can add aggregate functions to it. Use DISTINCT when you simply want unique values; use GROUP BY when you also need counts, sums, or other aggregations per group.
4

DISTINCT, AND, OR, NOT, BETWEEN, LIKE, and IN

Outcome 4
🧩

Outcome 4 · Logical & Range KeywordsCombining and Extending WHERE Conditions

AND

Both conditions must be TRUE. Narrows the result set.

OR

Either condition may be TRUE. Broadens the result set.

NOT

Inverts a condition. Returns rows where condition is FALSE.

BETWEEN

Tests if a value falls within an inclusive range (a AND b).

LIKE

Pattern matching using wildcards: % (any chars) and _ (one char).

IN

Tests if a value matches any item in a list or subquery.

AND and OR — Combining Conditions

T-SQL · AND / OR
-- AND: both conditions must be true
SELECT LastName, DeclaredMajor, GPA
FROM   Student
WHERE  DeclaredMajor = 'CIS'
   AND  GPA >= 3.50;
-- Returns: Adams (3.85), Chen (3.50), Flores (3.92)

-- OR: at least one condition must be true
SELECT LastName, DeclaredMajor, GPA
FROM   Student
WHERE  DeclaredMajor = 'CIS'
   OR   DeclaredMajor = 'CS';
-- Returns all CIS and CS students

-- Mixing AND and OR — use parentheses to control precedence
-- AND has higher precedence than OR — always use parens when mixing
SELECT LastName, DeclaredMajor, GPA
FROM   Student
WHERE  (DeclaredMajor = 'CIS' OR DeclaredMajor = 'CS')
   AND  GPA >= 3.00;
-- Without parens, AND would bind tighter and give wrong results

NOT — Negating a Condition

T-SQL · NOT
-- NOT reverses a condition
SELECT LastName, DeclaredMajor
FROM   Student
WHERE NOT DeclaredMajor = 'MIS';
-- Equivalent to: WHERE DeclaredMajor <> 'MIS'

-- NOT combined with other keywords
WHERE NOT GPA BETWEEN 3.0 AND 4.0   -- GPA outside 3.0–4.0
WHERE NOT LastName LIKE 'A%'        -- last name does NOT start with A
WHERE     DeclaredMajor NOT IN ('CIS', 'CS')   -- not CIS or CS

BETWEEN — Inclusive Range Testing

T-SQL · BETWEEN
-- BETWEEN is inclusive — includes both the lower and upper bounds
SELECT LastName, GPA
FROM   Student
WHERE  GPA BETWEEN 3.00 AND 3.90;
-- Equivalent to: WHERE GPA >= 3.00 AND GPA <= 3.90
-- Returns: Brown? No. Chen (3.50), Davis (3.10), Harris (3.40)

-- BETWEEN with dates
SELECT StudentID, LastName, CreatedDate
FROM   Student
WHERE  CreatedDate BETWEEN '2024-01-01' AND '2024-12-31';
-- Students who enrolled during 2024

-- NOT BETWEEN — outside the range
SELECT LastName, GPA
FROM   Student
WHERE  GPA NOT BETWEEN 2.00 AND 3.49;
-- Returns students with GPA below 2.00 or above 3.49

LIKE — Pattern Matching

T-SQL · LIKE — Wildcards % and _
-- % matches any sequence of zero or more characters
WHERE LastName LIKE 'A%'       -- starts with A  → Adams
WHERE LastName LIKE '%son'     -- ends with 'son'
WHERE LastName LIKE '%ar%'    -- contains 'ar'  → Harris

-- _ matches exactly ONE character
WHERE LastName LIKE 'C__n'     -- C + any 2 chars + n  → Chen
WHERE Phone    LIKE '305-___-____'  -- Miami area code format

-- Practical: find all emails from a university domain
SELECT LastName, Email
FROM   Student
WHERE  Email LIKE '%@fau.edu';

-- NOT LIKE — pattern does not match
SELECT LastName
FROM   Student
WHERE  LastName NOT LIKE '[AEIOU]%';  -- does not start with a vowel

IN — Matching Against a List

T-SQL · IN — List of Values
-- IN tests whether a value matches any item in a list
-- Much cleaner than multiple OR conditions
SELECT LastName, DeclaredMajor
FROM   Student
WHERE  DeclaredMajor IN ('CIS', 'CS');
-- Equivalent to: WHERE DeclaredMajor = 'CIS' OR DeclaredMajor = 'CS'

-- NOT IN — value not in the list
SELECT LastName, DeclaredMajor
FROM   Student
WHERE  DeclaredMajor NOT IN ('MIS');

-- IN with numbers
SELECT StudentID, LastName
FROM   Student
WHERE  StudentID IN (1, 3, 5, 7);

-- ⚠️  NOT IN with NULL — dangerous!
-- If the list contains NULL, NOT IN returns no rows
-- because: x NOT IN (..., NULL) = x <> NULL = UNKNOWN
WHERE  DeclaredMajor NOT IN ('MIS', NULL)  -- returns 0 rows!

Aggregate Functions — SUM, COUNT, MIN, MAX, AVG

Outcome 5
📊

Outcome 5 · AggregatesComputing Summary Statistics Across Rows

Aggregate functions collapse multiple rows into a single summary value. They operate on a set of rows and return one result per group (or one result for the entire table if no GROUP BY is used).

COUNT()

Counts the number of rows or non-NULL values

SUM()

Adds all numeric values in a column

AVG()

Calculates the arithmetic mean of numeric values

MIN()

Returns the smallest value (works on numbers, dates, text)

MAX()

Returns the largest value (works on numbers, dates, text)

COUNT — Counting Rows and Values

T-SQL · COUNT
-- COUNT(*) — counts all rows including NULLs
SELECT COUNT(*) AS TotalStudents
FROM   Student;
-- Returns: 8

-- COUNT(column) — counts only non-NULL values in that column
SELECT COUNT(DeclaredMajor) AS StudentsWithMajor
FROM   Student;
-- Returns: 8 (all have a major in our sample)
-- Would return less if some DeclaredMajor values were NULL

-- COUNT(DISTINCT column) — counts unique non-NULL values
SELECT COUNT(DISTINCT DeclaredMajor) AS NumberOfMajors
FROM   Student;
-- Returns: 3 (CIS, CS, MIS)

-- COUNT with WHERE — count rows matching a condition
SELECT COUNT(*) AS HighAchieverCount
FROM   Student
WHERE  GPA >= 3.50;
-- Returns: 3 (Adams, Chen, Flores)

SUM, AVG, MIN, MAX

T-SQL · SUM, AVG, MIN, MAX
-- All four in one query
SELECT
    COUNT(*) AS TotalStudents,
    AVG(GPA)  AS AverageGPA,
    MIN(GPA)  AS LowestGPA,
    MAX(GPA)  AS HighestGPA,
    SUM(GPA)  AS SumOfGPAs    -- not meaningful alone, but shows SUM syntax
FROM   Student;

-- AVG ignores NULL values — be careful if NULLs exist in the column
SELECT AVG(GPA) AS AvgGPA
FROM   Student
WHERE  DeclaredMajor = 'CIS';
-- Average GPA for CIS students only: (3.85 + 3.50 + 3.92) / 3 = 3.757

-- MIN and MAX work on text and dates too
SELECT
    MIN(LastName)    AS FirstAlphabetically,
    MAX(LastName)    AS LastAlphabetically,
    MIN(CreatedDate) AS EarliestEnrollment,
    MAX(CreatedDate) AS MostRecentEnrollment
FROM   Student;
TotalStudentsAverageGPALowestGPAHighestGPA
83.07751.953.92
(1 row — aggregate over all 8 students)
⚠️
Cannot Mix Aggregate and Non-Aggregate in SELECT Without GROUP BY You cannot write SELECT LastName, AVG(GPA) FROM Student — SQL cannot return 8 names and 1 average in the same row without a GROUP BY. If you mix regular columns and aggregates, every non-aggregate column must appear in the GROUP BY clause.

GROUP BY and HAVING — Aggregating by Category

Outcome 5
📂

Outcome 5 · GROUP BY & HAVINGGrouping Rows into Categories and Filtering Groups

GROUP BY partitions rows into groups based on shared values, then applies aggregate functions within each group independently. HAVING filters those groups — it is to GROUP BY what WHERE is to individual rows.

T-SQL · GROUP BY — Aggregates per Category
-- Count students and compute average GPA per major
SELECT
    DeclaredMajor,
    COUNT(*)  AS StudentCount,
    AVG(GPA)  AS AverageGPA,
    MIN(GPA)  AS LowestGPA,
    MAX(GPA)  AS HighestGPA
FROM   Student
GROUP BY DeclaredMajor
ORDER BY AverageGPA DESC;

-- Rule: every column in SELECT that is NOT an aggregate
--       MUST appear in GROUP BY
DeclaredMajorStudentCountAverageGPALowestGPAHighestGPA
CIS33.7573.503.92
MIS32.7672.203.40
CS22.5251.953.10
(3 rows — one per major)

HAVING — Filtering Groups

T-SQL · HAVING — WHERE for Groups
-- Only show majors with more than 2 students AND an average GPA above 3.0
SELECT
    DeclaredMajor,
    COUNT(*) AS StudentCount,
    AVG(GPA) AS AverageGPA
FROM   Student
GROUP BY DeclaredMajor
HAVING   COUNT(*) > 2
   AND   AVG(GPA) > 3.0
ORDER BY AverageGPA DESC;

-- WHERE vs HAVING — critical distinction:
--   WHERE   filters individual ROWS   before grouping
--   HAVING  filters GROUPS            after grouping

-- Using both WHERE and HAVING together
SELECT
    DeclaredMajor,
    COUNT(*) AS StudentCount
FROM   Student
WHERE  IsEnrolled = 1          -- filter to active students first (row-level)
GROUP BY DeclaredMajor
HAVING   COUNT(*) >= 2;       -- then keep only majors with 2+ active students
ClauseFiltersRunsCan use aggregates?
WHEREIndividual rowsBefore GROUP BY❌ No
HAVINGGroups of rowsAfter GROUP BY✅ Yes

Subqueries — Queries Inside Queries

Outcome 6
🪆

Outcome 6 · SubqueriesUsing a Query Result as a Filter Condition for Another Table

Definition — Subquery

A subquery (also called an inner query or nested query) is a SELECT statement embedded inside another SQL statement. The outer query uses the subquery's result as part of its WHERE condition, allowing you to filter one table based on data from another — without a JOIN.

Subquery in WHERE with IN — Most Common Pattern

T-SQL · Subquery with IN
-- GOAL: Find all students who are enrolled in section 101.
--        Enrollment table links StudentID to SectionID.
-- ─────────────────────────────────────────────────────────────────────────

-- Step 1 (inner query): get the StudentIDs enrolled in section 101
SELECT StudentID
FROM   Enrollment
WHERE  SectionID = 101;
-- Returns: 1, 3, 6

-- Step 2 (combined): use that result to filter the Student table
SELECT LastName, FirstName, GPA
FROM   Student
WHERE  StudentID IN
    (
        SELECT StudentID
        FROM   Enrollment
        WHERE  SectionID = 101
    );
-- Returns: Adams, Chen, Flores — all enrolled in section 101
LastNameFirstNameGPA
AdamsMaria3.85
ChenLi3.50
FloresAna3.92
(3 rows — students enrolled in section 101)

Subquery with NOT IN — Exclusion Pattern

T-SQL · Subquery with NOT IN — Find Students NOT Enrolled
-- Find students who are NOT enrolled in ANY section
SELECT LastName, FirstName
FROM   Student
WHERE  StudentID NOT IN
    (
        SELECT DISTINCT StudentID
        FROM   Enrollment
    );

-- ⚠️  NOT IN + NULL danger: if Enrollment.StudentID contains any NULL,
--     NOT IN returns 0 rows. Use NOT EXISTS instead for NULL safety:
SELECT LastName, FirstName
FROM   Student s
WHERE NOT EXISTS
    (
        SELECT 1
        FROM   Enrollment e
        WHERE  e.StudentID = s.StudentID
    );

Subquery with Comparison Operators — Single-Value Subqueries

T-SQL · Subquery Returning a Single Value
-- Find students whose GPA is above the overall average
-- Inner query returns ONE value (the average) → use = > < etc.
SELECT LastName, FirstName, GPA
FROM   Student
WHERE  GPA >
    (
        SELECT AVG(GPA)
        FROM   Student
    )
ORDER BY GPA DESC;
-- AVG(GPA) = 3.0775 — returns Adams (3.85), Chen (3.50), Flores (3.92), Harris (3.40)

-- Find the student with the highest GPA
SELECT LastName, FirstName, GPA
FROM   Student
WHERE  GPA =
    (
        SELECT MAX(GPA)
        FROM   Student
    );
-- Returns: Flores, Ana, 3.92

Subquery in SELECT — Scalar Subquery

T-SQL · Scalar Subquery in SELECT Column List
-- Show each student's GPA alongside the overall average for comparison
SELECT
    LastName,
    GPA,
    (
        SELECT AVG(GPA) FROM Student
    )                       AS SchoolAvgGPA,
    GPA - (
        SELECT AVG(GPA) FROM Student
    )                       AS DifferenceFromAvg
FROM   Student
ORDER BY GPA DESC;
💡
Subquery vs. JOIN — When to Use Each Use a subquery when you want to filter rows in one table based on data in another, and you don't need to display columns from both tables. Use a JOIN when you need to display columns from multiple tables in the same result. JOINs are often more readable and generally perform better on large datasets.

JOIN Operations — Combining Data from Multiple Tables

Outcome 7
🔗

Outcome 7 · JOIN TypesINNER JOIN, LEFT JOIN, RIGHT JOIN, and FULL OUTER JOIN Explained

A JOIN combines rows from two or more tables based on a related column — usually a foreign key matching a primary key. The join type determines which rows are included when one side has no match.

INNER JOIN

Returns only rows that have a match in both tables. Unmatched rows from either table are excluded.

Most common join type
LEFT (OUTER) JOIN

Returns all rows from the left table, plus matched rows from the right. Unmatched right-side values = NULL.

Find records with/without matches
RIGHT (OUTER) JOIN

Returns all rows from the right table, plus matched rows from the left. Unmatched left-side values = NULL.

Less common — LEFT JOIN preferred
FULL OUTER JOIN

Returns all rows from both tables. NULLs fill in wherever there is no match on either side.

Find all unmatched rows

INNER JOIN — The Most Common Join

T-SQL · INNER JOIN
-- INNER JOIN syntax: only rows where the ON condition matches appear in result
SELECT
    s.StudentID,
    s.LastName,
    s.FirstName,
    e.SectionID,
    e.Grade
FROM   Student    s              -- table alias 's' for brevity
INNER JOIN Enrollment e          -- table alias 'e'
    ON s.StudentID = e.StudentID  -- join condition: match on StudentID
ORDER BY s.LastName;

-- Students without any enrollment are EXCLUDED from INNER JOIN results
-- Enrollment rows with no matching student are also excluded

Three-Table INNER JOIN

T-SQL · Three-Table JOIN — Student + Enrollment + Section
-- Show student names, course ID, semester, and grade together
SELECT
    s.LastName,
    s.FirstName,
    sec.CourseID,
    sec.Semester,
    sec.YearOffered,
    e.Grade
FROM        Student    s
INNER JOIN  Enrollment e
    ON  s.StudentID   = e.StudentID
INNER JOIN  Section    sec
    ON  e.SectionID   = sec.SectionID
ORDER BY s.LastName, sec.Semester;

-- Each JOIN adds another table. Always chain on the FK = PK relationship.

T-SQL · LEFT JOIN — All Students, Including Unenrolled

-- Unenrolled students have NULL for enrollment columns

Using LEFT JOIN to Find Unmatched Rows — "Anti-Join" Pattern

T-SQL · LEFT JOIN Anti-Join — Find Students with NO Enrollments
-- Find students who are NOT enrolled in any section
-- Filter WHERE right-side column IS NULL after LEFT JOIN
SELECT
    s.StudentID,
    s.LastName,
    s.FirstName
FROM       Student    s
LEFT JOIN  Enrollment e
    ON  s.StudentID = e.StudentID
WHERE      e.StudentID IS NULL;
-- Only rows where no Enrollment match was found → unenrolled students

FULL OUTER JOIN — All Rows from Both Tables

T-SQL · FULL OUTER JOIN
-- Returns all students AND all enrollment records
-- NULLs appear where no match exists on either side
SELECT
    s.LastName,
    s.FirstName,
    e.SectionID,
    e.Grade
FROM            Student    s
FULL OUTER JOIN Enrollment e
    ON  s.StudentID = e.StudentID;
-- Use case: data auditing — find unmatched rows on EITHER side

JOIN with WHERE, GROUP BY, and ORDER BY — Full Query

T-SQL · Full Multi-Clause Query with JOIN
-- Report: for each CIS student, show how many sections they are enrolled in
--          and their average grade — sorted by enrollment count descending
SELECT
    s.LastName,
    s.FirstName,
    s.GPA,
    COUNT(e.SectionID)  AS SectionsEnrolled,
    AVG(
        CASE e.Grade
            WHEN 'A'  THEN 4.0
            WHEN 'A-' THEN 3.7
            WHEN 'B+' THEN 3.3
            WHEN 'B'  THEN 3.0
            ELSE NULL
        END
    )                    AS AvgGradePoints
FROM       Student    s
LEFT JOIN  Enrollment e  ON  s.StudentID = e.StudentID
WHERE      s.DeclaredMajor = 'CIS'
GROUP BY   s.StudentID, s.LastName, s.FirstName, s.GPA
HAVING     COUNT(e.SectionID) > 0
ORDER BY   SectionsEnrolled DESC;

ClauseA keyword-introduced component of a SELECT statement (FROM, WHERE, ORDER BY, etc.)
PredicateA condition in a WHERE or HAVING clause that evaluates to TRUE, FALSE, or UNKNOWN
NULLThe absence of a value — not zero, not empty string; comparisons with NULL return UNKNOWN
AliasA temporary name assigned to a column or table using AS — improves readability
Aggregate functionA function (COUNT, SUM, AVG, MIN, MAX) that collapses multiple rows into a single summary value
GROUP BYClause that partitions rows into groups for per-group aggregation
HAVINGClause that filters groups after GROUP BY; can reference aggregate functions
SubqueryA SELECT statement nested inside another statement; inner result used by outer query
Correlated subqueryA subquery that references columns from the outer query; evaluated once per outer row
JOINOperation that combines rows from two or more tables based on a related column condition
INNER JOINReturns rows with a matching value in both tables; excludes non-matching rows
LEFT JOINReturns all rows from the left table plus matching rows from the right; NULLs where no match
Anti-joinPattern using LEFT JOIN + WHERE IS NULL to find rows in one table with no match in another
Table aliasA short name given to a table in a query (e.g. Student s) for cleaner column references
WildcardSpecial LIKE characters: % (any sequence) and _ (exactly one character)

Self-Assessment Checklist

✅
  • I can explain the difference between DQL and other SQL sub-languages (DDL, DML, DCL)
  • I understand the SELECT statement's write order vs. execution order and why it matters
  • I can write a basic SELECT query using SELECT, FROM, and WHERE to retrieve and filter rows from a single table
  • I can use comparison operators (=, <>, >, <, >=, <=) and IS NULL / IS NOT NULL in WHERE conditions
  • I can sort query results using ORDER BY with ASC and DESC, including multi-column sort
  • I can use DISTINCT to remove duplicate values from a result set
  • I can combine conditions using AND, OR, and NOT — and I know to use parentheses when mixing AND and OR
  • I can use BETWEEN for inclusive range checks and understand it is equivalent to >= AND <=
  • I can use LIKE with % and _ wildcards to perform pattern matching on string columns
  • I can use IN to match values against a list, and I know why NOT IN with NULLs is dangerous
  • I can use COUNT, SUM, AVG, MIN, and MAX aggregate functions and understand how each handles NULLs
  • I can write a GROUP BY query and include all non-aggregate SELECT columns in the GROUP BY clause
  • I can use HAVING to filter aggregated groups and I know the difference between WHERE and HAVING
  • I can write a subquery using IN to filter one table based on values from another table
  • I can write INNER JOIN and LEFT JOIN queries, using table aliases and ON conditions
  • I can use the LEFT JOIN anti-join pattern (WHERE right.col IS NULL) to find unmatched rows
  • I can combine JOIN with WHERE, GROUP BY, HAVING, and ORDER BY in a single query
